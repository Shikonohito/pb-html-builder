Заданные ограничения:

1. Инструмент ориентирован на Windows.

2. Инструмент локальный, одиночный, с одной открытой вкладкой.

3. Инструмент должен позволять собирать определённые HTML-страницы по принципу WYSIWYG.

4. Стили и скрипты подтягиваются с github pages (на время разработки файлы локальные).



Дополнительный контекст:

1. Бэкенд - C#.

2. Фронтенд - Blazor.

3. Генерируемые HTML-файлы предназначены для студентов, которые изучают Python.

4. HTML-файлы должны быть лёгкими, так как их будут часто пересылать по сети.

5. Есть только три вида HTML-файлов: theory, practice и assessment.

6. Файлы theory и practice не хранят данные между сессиями. Это значит при обновлении этих страниц изменения не сохраняются. Файл assessment хранит данные между сессиями.

7. Для отображения и редактирования блоков кода планируется Monaco Editor.

8. Для запуска Python-кода планируется Pyodide.

9. Для загрузки/сохранения важны метаданные внутри HTML-файла.



Требования к стартовой странице builder:

1. На стартовой странице в центре есть блок с кнопками – New Theory, New Practice, New Assessment, New Verified Assessment.

2. При нажатии на New Theory, New Practice New Assessment, New Verified Assessment, происходит плавный сдвиг справа налево текущего блока на блок создания.

3. Для New Theory, New Practice, New Assessment и New Verified Assessment в блоке создания можно указать путь сохранения через ручной ввод или через кнопку указания пути. Ниже поле для ввода названия файла и ещё ниже поле для ввода заголовка (будет отображаться на вкладке браузера и в h1 заголовке верхней панели во время builder). Ниже две кнопки – `Create` и `Cancel`. При нажатии на `Cancel`, происходит плавный сдвиг слева направо текущего блока на стартовый блок. При нажатии `Create`, от этой кнопки происходит плавный круговой переход в виде волны к Builder соответствующего html файла – Theory, Practice, Assessment.

4. Для New Verified Assessment в блоке создания дополнительно нужно указать путь к существующему html-assessment файлу и путь к существующему json файлу с ответами.



Требования к html-theory:

1. Файл html состоит из верхней панели, боковой панели и главной части.

2. Верхняя панель содержит кнопку показа/скрытия боковой панели, правее заголовок темы, который состоит из topic-kicker с текстом `Python Bootcamp` и ниже h1 заголовок с текстом темы, который builder позволяет изменять. Ещё правее в самой правой части находится кнопка смены светлой/тёмной темы.

3. Боковая панель содержит заголовок, в которой находится текст `Оглавление` и правее кнопка скрытия боковой панели. Ниже находится блок со списком оглавлений, которые дублируют порядковые номера и заголовки секций из блока уроков. В самом низу находится текст `© Ivan Ivanov, 2026`.

4. Главная часть состоит из карты секций и блоком урок.

5. Карта секций состоит из левой и правой части. Левая часть состоит из текста `Разделы темы` и ниже текст `Что важно рассмотреть`. Правая часть состоит из сетки с двумя колонками и тремя рядами, в которых находятся заголовки с якорями. Builder должен позволять изменять текст и привязывать якоря.

6. Блок уроков состоит из секций. Секция должна иметь заголовок и тело. Заголовок состоит из порядкового номера в формате `01` и h2 текста, который builder позволяет изменять.

7. Секция может содержать параграфы, нумерованные списки, маркированные списки, таблицы, встроенные картинки, встроенные видео, специальные блоки (определение термина, важный момент, подсказка, верный подход, неверный подход), блок Monaco Editor (readonly, readonly и run, edit и run), блок вопросов (с несколькими вариантами и одним верным ответом, с несколькими вариантами и несколькими верными ответами, только с полем ввода ответа), которые редактор позволяет добавлять или удалять.

8. Параграфы могут содержать обычный текст, жирный, курсив, подчёркнутый, inline code, ссылки. Builder должен позволять менять текст параграфов, делать жирным, курсивом, подчёркнутым, inline code, ссылки.

9. Специальные блоки строятся по одному шаблону, но различаются наполнением и стилем. Есть block-label, ниже h3 заголовок блока и ниже тело блока. block-label может быть одним из вариантов – определение термина, важный момент, подсказка, верный подход, неверный подход. Заголовок блока h3 может содержать обычный текст и inline code, который builder позволяет изменять. Тело блока может содержать параграфы, нумерованные списки, маркированные списки, таблицы, встроенные картинки, встроенные видео, блок Monaco Editor (readonly, readonly и run, edit и run).

10. Блок Monaco Editor состоит из трёх частей – заголовка (редактируемый), кнопок действий (reset, run, copy) и редактор с кодом. Если кнопка run активна, то дополнительно ниже редактора находится блок вывода. Builder должен позволять включать/выключать кнопки действий, а также указывать предустановленный код в редакторе.

11. Блок вопросов похож на специальный блок – есть block-label, ниже h3 заголовок блока и ниже тело блока. block-label может быть одним из вариантов – вопрос с одним ответом, вопрос с несколькими ответами, открытый вопрос. Заголовок блока h3 может содержать обычный текст и inline code, который builder позволяет изменять. Тело блока может содержать параграфы, нумерованные списки, маркированные списки, таблицы, встроенные картинки, встроенные видео, блок Monaco Editor (readonly, readonly и run).

12. Блок вопроса с одним ответом или блок вопроса с несколькими ответами в самом низу содержит блок вариантов ответа. Варианты ответов содержат текст. Текст может содержать обычный, жирный, подчёркнутый, inline code. При нажатии на неверный или верный ответ, под вариантом ответа плавно появляется feedback. Builder должен позволять изменять количество вариантов ответов, менять текст вариантов, изменять feedback и указывать, какие варианты верные, а какие нет.

13. Блок с открытым вопросом в самом низу содержит блок ответа. Блок ответа содержит заголовок `Ваш ответ`, поле для ввода и правее кнопка `Ответить`. При нажатии на кнопку `Ответить` под ответом появляется feedback, в зависимости от верного или неверного ответа. Builder должен позволять изменять feedback и указывать, какие варианты верные.



Требования к builder при создании html-theory:

1. На верхней панели при изменении h1 заголовка меняется заголовок вкладки браузера.

2. Если секций нет, то в главной части есть кнопка-кружок с svg иконкой плюса внутри, при нажатии на который создаётся секция.

3. Если секции есть, то можно навести курсор над или под секцию, чтобы появилась горизонтальная пунктирная линия с кнопкой-кружок с svg иконкой плюса внутри, при нажатии на который создаётся секция в соответствующем месте. Также можно навести курсор над или под элемент боковой панели, чтобы появилась горизонтальная пунктирная линия с кнопкой-кружок с svg иконкой плюса внутри, при нажатии на который создаётся новый элемент боковой панели и новая секция в главной части.

4. Чтобы удалить секцию, нужно нажать правую кнопку мыши по секции или соответствующему элементу из боковой панели и нажать кнопку с svg иконкой мусорки.

5. Чтобы переместить секцию нужно зажать левую кнопку мыши на секции и тянуть вверх или вниз. Когда курсор с секцией находится над/под половиной секции, то появляется призрак перетаскиваемой секции над/под другой секцией.

6. Чтобы переместить секцию, можно ещё зажать левую кнопку мыши на элементе боковой панели и тянуть вверх или вниз. Соответствующая секция изменит положение.

7. При создании/удалении/перемещении секций, элементы боковой панели автоматически отображают тот же порядок, что и порядок секций.

8. При изменении заголовка секций, соответствующий элемент боковой панели так же изменяет своё название.

9. При наведении курсора на секцию, элементы секции или элементы боковой панели, показываются границы элемента, чтобы было явно видно, с чем будет взаимодействовать контекстное меню.

10. После создания секции в теле секции показывается кнопка-кружок с svg иконкой плюса, при нажатии на которую показывается меню из доступных элементов, которые можно добавить – параграф, нумерованный список, маркированный список, таблица, встроенные картинки, встроенные видео, специальный блок, блок Monaco Editor, блок вопросов.

11. При добавлении встроенной картинки или встроенного видео появляется поле для ввода ссылки. Если нажать клавишу `Esc`, то добавление отменяется. Если ввести ссылку и нажать клавишу `Enter`, то картинка или видео встраивается в место добавления.

12. Если навести курсор над/под параграф, нумерованный список, маркированный список, таблицу, встроенные картинки, встроенные видео, специальный блок, блок Monaco Editor или блок вопросов, то появляется горизонтальная пунктирная линия с кнопкой-кружком с svg иконкой плюса внутри. Если нажать кнопку, то показывается меню из доступных элементов, которые можно добавить – параграф, нумерованный список, маркированный список, таблица, специальный блок, блок Monaco Editor, блок вопросов.

13. Если нажать правую кнопку мыши на параграф, нумерованный список, маркированный список, таблицу, встроенные картинки, встроенные видео, специальный блок, блок Monaco Editor или блок вопросов, то появляется контекстное меню, в котором можно выбрать кнопку с svg иконкой мусорки, чтобы удалить.

Если зажать левую кнопку мыши на параграфе, нумерованном списке, маркированном списке, таблице, встроенные картинки, встроенные видео, специальном блоке, блоке Monaco Editor или блоке вопросов, то можно потянуть вверх/вниз, чтобы переместить. Когда курсор с элементом находится над/под половиной другого элемента, то появляется призрак перетаскиваемого элемента над/под другим элементом.

16. В специальном блоке если тело пустое, то можно нажать на кнопку-кружок с svg иконкой плюса, при нажатии на которую показывается меню из доступных элементов, которые можно добавить – параграф, нумерованный список, маркированный список, таблица, встроенные картинки, встроенные видео, блок Monaco Editor.

Неактивная кнопка визуально выглядит как прозрачная с пунктирными границами. Активная – непрозрачная и solid границы. Кнопка run по умолчанию неактивная. Кнопка readonly по умолчанию активная.

В итоговом html файле в блоке Monaco Editor в правом верхнем углу слева направо должны быть кнопки – reset (svg иконка круговой стрелочки; показывается, если кнопка readonly неактивна), run (если кнопка run активна), copy (svg иконка вертикального прямоугольника на вертикальном прямоугольнике).

19. В блоке Monaco Editor чтобы сделать python код по умолчанию, достаточно просто написать python код внутри редактора Monaco Editor. В итоговом html файле этот код будет вставляться в редактор при нажатии на кнопку reset.

20. В блоке вопросов если тело пустое, то можно нажать на кнопку-кружок с svg иконкой плюса, при нажатии на которую показывается меню из доступных элементов, которые можно добавить – параграф, нумерованный список, маркированный список, таблица, встроенные картинки, встроенные видео, блок Monaco Editor.

Чтобы удалить вариант, достаточно нажать правую кнопку мыши по варианту и выбрать кнопку с svg иконкой мусорки. Под последним вариантом находится широкая кнопка с пунктирными границами и svg иконкой плюса в середине, при нажатии на которую появляется новый вариант. При наведении курсора на блок вариантов, внутри вариантов в правой части появляется прозрачная круглая кнопка-переключатель (верный/неверный вариант) и варианты окрашиваются в цвета (зелёный оттенок для верного и красный оттенок для неверного). Верный вариант это галочка, неверный вариант это крестик. Если верных вариантов больше одного, то текст в block-label заменяется на `Вопрос с несколькими ответами`. У каждого варианта в правой части есть кнопка создания фидбэка (svg иконка прямоугольника сообщения), при нажатии на который под вариантом появляется блок фидбэка, куда можно вписать текст.

Можно выделить текст в заголовке Monaco Editor, h3 заголовок блока вопросов и h3 заголовок специального блока, и увидеть над выделенным поверх всего маленькую панель с кнопками – сделать inline code. Ещё можно нажать правую кнопку мыши по выделенному, чтобы открыть контекстное меню, и выбрать те же действия, которые расположены в виде списка. 9. При наведении курсора на секцию, элементы секции или элементы боковой панели, показываются границы элемента, чтобы было явно видно, с чем будет взаимодействовать контекстное меню.

10. После создания секции в теле секции показывается кнопка-кружок с svg иконкой плюса, при нажатии на которую показывается меню из доступных элементов, которые можно добавить – параграф, нумерованный список, маркированный список, таблица, встроенные картинки, встроенные видео, специальный блок, блок Monaco Editor, блок вопросов.

11. При добавлении встроенной картинки или встроенного видео появляется поле для ввода ссылки. Если нажать клавишу `Esc`, то добавление отменяется. Если ввести ссылку и нажать клавишу `Enter`, то картинка или видео встраивается в место добавления.

12. Если навести курсор над/под параграф, нумерованный список, маркированный список, таблицу, встроенные картинки, встроенные видео, специальный блок, блок Monaco Editor или блок вопросов, то появляется горизонтальная пунктирная линия с кнопкой-кружком с svg иконкой плюса внутри. Если нажать кнопку, то показывается меню из доступных элементов, которые можно добавить – параграф, нумерованный список, маркированный список, таблица, специальный блок, блок Monaco Editor, блок вопросов.

13. Если нажать правую кнопку мыши на параграф, нумерованный список, маркированный список, таблицу, встроенные картинки, встроенные видео, специальный блок, блок Monaco Editor или блок вопросов, то появляется контекстное меню, в котором можно выбрать кнопку с svg иконкой мусорки, чтобы удалить.

14. Если зажать левую кнопку мыши на параграфе, нумерованном списке, маркированном списке, таблице, встроенные картинки, встроенные видео, специальном блоке, блоке Monaco Editor или блоке вопросов, то можно потянуть вверх/вниз, чтобы переместить. Когда курсор с элементом находится над/под половиной другого элемента, то появляется призрак перетаскиваемого элемента над/под другим элементом.

15. У специального блока можно нажать левую кнопку мыши на block-label, чтобы открыть выпадающий список доступных вариантов. Выбранный вариант меняет стиль.

16. В специальном блоке если тело пустое, то можно нажать на кнопку-кружок с svg иконкой плюса, при нажатии на которую показывается меню из доступных элементов, которые можно добавить – параграф, нумерованный список, маркированный список, таблица, встроенные картинки, встроенные видео, блок Monaco Editor.

17. В блоке Monaco Editor при наведении курсора на область с кодом появляются кнопки в правой верхней части слева направо – run (svg иконка play) и readonly (svg иконка lock). Неактивная кнопка визуально выглядит как прозрачная с пунктирными границами. Активная – непрозрачная и solid границы. Кнопка run по умолчанию неактивная. Кнопка readonly по умолчанию активная. На одну секцию может быть только один блок Monaco Editor с активной кнопкой run.

18. В блоке Monaco Editor если кнопку run сделать активным, то под блоком вывода появляется блок редактирования автотестов. В этом блоке находится широкая прямоугольная кнопка с пунктирными границами и svg иконкой плюса в центре. При нажатии на эту кнопку появляются варианты – input/print, function, class. При выборе input/print появляется два поля для ввода – слева ввод, справа ожидаемый вывод. Множественный ввод можно производить с указанием \n. При выборе function появляется одно поле для ввода, куда можно вписать по формату `func_name(*args) == expected_value,`. При выборе class появляется блок Monaco Editor, в который можно вписать сценарный Python-тест. Все автотесты имеют нумерацию в левой части. При сохранении автотесты записываются в специальный общий json файл.

19. В блоке Monaco Editor если кнопку run сделать активным, то под автотестами появляется отдельный Monaco Editor, куда можно вписать ожидаемый ответ. При сохранении этот код записывается в специальный общий json файл.

20. В итоговом html файле в блоке Monaco Editor в правом верхнем углу слева направо должны быть кнопки – reset (svg иконка круговой стрелочки; показывается, если кнопка readonly неактивна), run (если кнопка run активна), copy (svg иконка вертикального прямоугольника на вертикальном прямоугольнике).

21. В блоке Monaco Editor чтобы сделать python код по умолчанию, достаточно просто написать python код внутри редактора Monaco Editor. В итоговом html файле этот код будет вставляться в редактор при нажатии на кнопку reset.

22. В блоке вопросов если тело пустое, то можно нажать на кнопку-кружок с svg иконкой плюса, при нажатии на которую показывается меню из доступных элементов, которые можно добавить – параграф, нумерованный список, маркированный список, таблица, встроенные картинки, встроенные видео, блок Monaco Editor.

23. В блоке вопросов если нажать на block-label, то можно выбрать Single-Multiple Answer или Text Input Answer. При выборе Single-Multiple Answer текст в block-label заменяется на `Вопрос с одним ответом`. При выборе Text Input Answer текст в block-label заменяется на `Открытый вопрос`.

24. В блоке вопросов с Single-Multiple Answer в самом низу находится блок вариантов (по умолчанию 4). Чтобы удалить вариант, достаточно нажать правую кнопку мыши по варианту и выбрать кнопку с svg иконкой мусорки. Под последним вариантом находится широкая кнопка с пунктирными границами и svg иконкой плюса в середине, при нажатии на которую появляется новый вариант. При наведении курсора на блок вариантов, внутри вариантов в правой части появляется прозрачная круглая кнопка-переключатель (верный/неверный вариант) и варианты окрашиваются в цвета (зелёный оттенок для верного и красный оттенок для неверного). Верный вариант это галочка, неверный вариант это крестик. Если верных вариантов больше одного, то текст в block-label заменяется на `Вопрос с несколькими ответами`. При сохранении верный вариант записывается в специальный общий json файл.

25. В блоке вопросов с Text Input Answer в самом низу блок ответа. В блоке ответа есть поле ввода и правее кнопка ответить. Ещё ниже есть поля для ввода верных ответов (по умолчанию одно поле). Ниже последнего поля для ввода верных ответов показывается прямоугольная кнопка с пунктирными границами и svg иконкой плюса в центре, при нажатии на который появляется новое поле для ввода верного ответа. Эти поля нужны, чтобы указать варианты верных ответов. При нажатии правой кнопкой мыши по варианту верного ответа, можно нажать кнопку с svg иконкой мусорки, чтобы удалить его. При сохранении верные варианты записывается в специальный общий json файл.

26. Можно выделить текст в параграфе, нумерованном списке, маркированном списке, таблице, текст в вариантах ответов и текст в фидбэках в блоках вопросов, и увидеть над выделенным поверх всего маленькую панель с кнопками – сделать жирным, курсивным, подчёркнутым, inline code, привязать ссылку. Ещё можно нажать правую кнопку мыши по выделенному, чтобы открыть контекстное меню, и выбрать те же действия, которые расположены в виде списка. Если стиль применён, то кнопка "вдавлена", а в контекстном меню в самой правой части стоит галочка. При нажатии на кнопку привязки ссылки появляется поле для ввода ссылки. Аналогично можно нажать горячие клавиши. Для жирного Ctrl + B. Для курсивного Ctrl + I. Для подчёркнутого Ctrl + U. Для inline code Ctrl + Q. Для ссылки Ctrl + K. Также можно просто поставить курсор на слово и прожать горячие клавиши, чтобы применить стили для слова.

27. Можно выделить текст в заголовке Monaco Editor, h3 заголовок блока вопросов и h3 заголовок специального блока, и увидеть над выделенным поверх всего маленькую панель с кнопками – сделать inline code. Ещё можно нажать правую кнопку мыши по выделенному, чтобы открыть контекстное меню, и выбрать те же действия, которые расположены в виде списка. Если стиль применён, то кнопка "вдавлена", а в контекстном меню в самой правой части стоит галочка.

28. Должен быть стек изменений. При нажатии Ctrl + Z отменяется последнее действие. Чтобы вернуть действие, то нужно нажать Ctrl + Shift + Z.

29. Каждые 5 минут автосохранение – 10 слотов. Если все слоты используются, то перезаписываются самые старые слоты. При автосохранении появляется плашка сверху с надписью об автосохранении.

30. Чтобы сохранить текущую работу, можно нажать правую кнопку мыши, в контекстном меню навести курсор на Project, затем выбрать `Save` или `Save as...`. Пункт `Save` сохраняет в существующий файл, если путь установлен. Пункт `Save as...` открывает диалоговое окно для указания пути/файла для сохранения.

31. Чтобы открыть существующий проект, можно нажать правую кнопку мыши, в контекстном меню навести курсор на Project, затем выбрать `Open...`, при нажатии которого открывается диалоговое окно выбора html файла. Builder должен открывать только в том случае, если у файла есть определённые метаданные.



Требования к html-verified-assessment:

1. Полностью повторяет html-assessment, но только в формате readonly – нельзя выбирать вариант ответа, нельзя вводить ответ, нельзя вводить код в блок Monaco Editor.

2. В самом верху главной части большими символами указано соотношение `верные ответы / всего вопросов`.

3. В секциях вопросов неверные горят красными, верные – зелёными. Ниже может быть блок комментарий.

4. Секция вопросов с вводом под введённым вариантом указан список из верных вариантов. Ниже может быть блок комментарий

5. Секция с кодом сверху вниз – ваш код, результаты автотестов, ожидаемый код, блок комментарий.

6. Рамка секции красная, соответствующий элемент боковой панели имеет статус `Неверно`, если ответ неверный. Рамка секции зелёная, соответствующий элемент боковой панели имеет статус `Верно`, если ответ верный. Верные ответы проверяются через json файл (для вопросов) и через автотесты (для кода).



Требования к builder при создании html-verified-assessment:

1. При создании html-verified-assessment builder запускает проверку соответствий ответов, данных в html-assessment файле и json файле. Если ответ не найден, помечает секцию, как `Ожидает проверки`.

2. Позволяет добавить специальный блок с комментарием в секцию, отметить секцию, как неверный/верный ответ. Остальное неизменяемо.
