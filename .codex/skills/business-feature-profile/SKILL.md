# Business Feature Profile

## Правила процесса

Основная задача — управлять переходами между стадиями, передавать контекст между субагентами, показывать пользователю краткие итоги каждой стадии.

1. Каждая задача обязана проходить через стадии.
2. Выполнять стадии только через разрешённые переходы.
3. Перед каждым переходом явно сообщать: `Стадия <текущая> → <следующая>`.
4. Каждую рабочую стадию делегировать отдельному subagent.
5. Стадию Report выполняет оркестратор.
6. При запуске subagent передавать исходный запрос, актуальное описание бизнес-фичи, acceptance criteria, краткое резюме предыдущей стадии и причину отката.
7. Сохранять результат стадии как компактное резюме и передавать его дальше.

## Стадии и subagents

1. Research — исследование задачи, кодовой базы, зависимостей. subagent_type: `researcher`.
2. Plan — формирование плана реализации. subagent_type: `planner`.
3. Implement — осуществление плана. subagent_type: по стеку проекта (по инвариантам стека).
4. Validate — проверка, что фича работает и нет регрессий. subagent_type: `validator`.
5. Report — отчёт о проделанной работе. Занимается оркестратор..

## Разрешённые переходы

- Research → Plan.
- Research → Implement, только если задача мала, решение однозначно и отдельный план не добавит ценности; явно зафиксировать причину пропуска.
- Plan → Implement.
- Implement → Validate.
- Implement → Research, если обнаружены неизвестные зависимости или неверные предпосылки.
- Validate → Report.
- Validate → Implement, если реализация или тесты требуют корректировки.
- Validate → Research, если проверка опровергла исходные требования или архитектурные предпосылки.

## Критерии стадий

### Research

Получить сводку консилиума по архитектуре, frontend, UX, security и DevOps от subagent `researcher`. Зафиксировать затронутые модули, зависимости, ограничения, риски и открытые вопросы.

### Plan

Получить исполнимый план с файлами или модулями, последовательностью изменений, тестовой стратегией, миграциями, критериями готовности и всеми ключевыми архитектурными решениями от subagent `planner`.

### Implement

Делегировать реализацию согласованного scope отдельному subagent по стеку проекта. Зафиксировать изменённые файлы.

### Validate

Делегировать проверку исходного сценария, связанные регрессии, сборку и релевантные тесты отдельному subagent `validator`.

### Report

Создать `reports/feature-<slug>-report.html` на русском языке. Включить:

- Название фичи и дата
- Краткое описание задачи
- Итоги Research (сводка консилиума)
- План (из стадии Plan)
- Что реализовано (файлы, модули)
- Результаты Validate (тесты, платформы)
- Проблемы и откаты (если были)
- Статус `Done` или `Partially Done`
